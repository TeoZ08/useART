# Fase 2 proposta

## Objetivo

Substituir as persistências transitórias e validações locais por infraestrutura real, mantendo a experiência visual e os contratos criados na Fase 1.

## Escopo recomendado

- Supabase Auth para painel administrativo.
- Postgres para produtos, variações, pedidos, cupons e status.
- Supabase Storage para imagens.
- Mercado Pago Checkout Pro.
- Cálculo ou tabela real de frete.
- Webhooks de pagamento e atualização de pedido.
- Área operacional para acompanhar solicitações.

## Critério mínimo antes da Fase 2

- Confirmar imagens e dados comerciais pendentes.
- Aprovar textos jurídicos.
- Definir regras de cupom.
- Definir processo de atendimento e pagamento.
